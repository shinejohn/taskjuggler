# User Manuals and UI Mockups

This directory contains comprehensive user manuals and visual UI mockups for all four applications in the Task Juggler platform.

## User Manuals

### 1. TaskJuggler (`USER_MANUAL_TASKJUGGLER.md`)
Complete guide to task management and collaboration platform including:
- Dashboard overview
- Task creation and management
- Inbox processing
- Routing rules
- Teams and collaboration
- Messages and channels
- Marketplace
- Contact management

### 2. 4process.ai (`USER_MANUAL_4PROCESS.md`)
Guide to process automation and workflow management:
- Process Builder (visual workflow designer)
- Process templates
- Execution tracking
- Process settings and triggers

### 3. 4projects.ai (`USER_MANUAL_4PROJECTS.md`)
Project management platform documentation:
- Project creation and management
- Kanban board usage
- Timeline/Gantt view
- Task management
- Team collaboration

### 4. SiteHealth Scanner (`USER_MANUAL_SCANNER.md`)
Website health monitoring guide:
- Site management
- Scanning websites
- Issue detection and management
- AI-powered fix generation
- Reports and analytics

## UI Mockups

Visual HTML mockups are located in the `ui-mockups/` directory. Open these files in a web browser to see visual representations of key screens:

### TaskJuggler
- `taskjuggler-dashboard.html` - Main dashboard view
- `taskjuggler-tasks.html` - Task list view

### 4process.ai
- `4process-builder.html` - Visual process builder interface

### 4projects.ai
- `4projects-kanban.html` - Kanban board view

### SiteHealth Scanner
- `scanner-dashboard.html` - Scanner dashboard with site cards
- `scanner-issue.html` - Issue detail view with fix preview

## Usage

1. **Reading Manuals**: Open the markdown files in any markdown viewer or text editor
2. **Viewing Mockups**: Open HTML files directly in a web browser
3. **Printing**: Manuals can be printed or exported to PDF for offline use

## Design System

All applications use the **Fibonacco Design System v1.0** with:
- Modern glassmorphism aesthetic
- Consistent color palette
- Responsive design principles
- Accessibility-first approach

## Updates

These manuals are maintained alongside the applications. When features are added or changed, the manuals should be updated accordingly.
